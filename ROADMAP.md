# 개발 로드맵: Roguelike SRPG

## 프로젝트 구조 (확정)

```
srpg/
├── docs/
│   ├── GAME_DESIGN.md
│   └── plans/
├── core/           # 게임 루프, 윈도우, 카메라, DrawTile
├── map/            # 맵 타일, 맵 생성, 층 관리
├── entity/         # Entity/Operator/Party 순수 데이터 구조 및 초기화, 능력치 보너스 함수
├── battle/         # 전투 계산, 스킬, AI, 상태이상 처리
├── item/           # 아이템, 장비, 유물, 상점
├── ui/             # 렌더링, 로그, 플로팅 텍스트
└── main.c
```

### 도메인 책임 경계

| 도메인 | 포함 | 제외 |
|---|---|---|
| `core` | 게임 루프, 윈도우, 카메라, `DrawTile` | 게임 로직 일체 |
| `map` | 타일 정의, 맵 생성, 층 관리 (`FloorManager`) | 엔티티 배치 |
| `entity` | 구조체 정의, `InitEntity`, 능력치 보너스 함수 | 상태이상 처리, 전투 로직 |
| `battle` | 전투 계산, 스킬, AI, `UpdateStatusEffects` | 아이템 데이터 |
| `item` | 아이템/장비/유물 데이터, 상점 | 장착 효과 적용 (battle에서 처리) |
| `ui` | 렌더링, 로그, 플로팅 텍스트 | 게임 상태 관리 |

## 전체 개발 로드맵

### Phase 1: 기초 인프라 및 렌더링 (Foundation)

**목표**: raylib 윈도우 설정, 게임 루프, 텍스트 기반 타일 렌더링 구현.

**작업 범위**:
- `core/`: 윈도우 생성, 게임 루프, `GameCamera`, `DrawTile`
- `ui/`: 기본 로그 패널 렌더링
- `main.c`: 진입점, 루프 연결

**완료 기준**: 빈 맵(전체 바닥 타일)이 화면에 렌더링되고, 로그 패널이 하단에 표시된다.

---

### Phase 2: 맵 생성 및 엔티티 기초 (World & Entities)

*Phase 2와 3 통합: 맵과 엔티티는 상호 검증이 필요하므로 함께 구현한다.*

**목표**: 무작위 맵 생성 및 플레이어 엔티티의 그리드 기반 이동 구현.

**작업 범위**:
- `map/`: BSP 알고리즘 기반 무작위 방/복도 생성, 벽 충돌 판정
- `entity/`: `Entity`, `Operator`, `Party` 구조체 정의 및 초기화
- 플레이어 입력 처리 (방향키 이동), 카메라 추적

**완료 기준**: 무작위로 생성된 맵 위에서 `@` 기호가 벽을 통과하지 않고 이동한다.

---

### Phase 3: 전투 시스템 기초 (Combat Basics)

**목표**: 적 엔티티 배치 및 기본 전투 로직 구현.

**작업 범위**:
- `entity/`: 적 엔티티 초기화
- `battle/`: 물리/마법 데미지 공식, 명중/회피 계산, 사거리 판정
- 턴 교체 로직 (플레이어 턴 → 적 턴)
- `ui/`: 플로팅 데미지 텍스트

**완료 기준**: 플레이어가 적을 선택하여 공격할 수 있고, 데미지와 MISS가 화면에 표시된다. 적이 사망하면 맵에서 제거된다.

---

### Phase 4: 적 AI (Enemy AI)

**목표**: 적의 자율적 행동 패턴 구현.

**작업 범위**:
- `battle/`: 거리 기반 타겟 선정, 이동 및 공격 판정, 벽 우회 이동
- 상태이상 처리 (`UpdateStatusEffects`): 화상(Burn) 등 턴 종료 시 효과

**완료 기준**: 적이 턴마다 가장 가까운 플레이어를 향해 이동하고, 사거리 내에서 공격한다.

---

### Phase 5: 성장 및 능력치 시스템 (Growth & Stats)

**목표**: 레벨업, 경험치, 5대 기관 특수 능력치 구현.

**작업 범위**:
- `entity/`: 능력치 보너스 함수 (`GetEyeBonus`, `GetHandBonus` 등)
- `battle/`: 레벨업 시 보너스 적용 로직
- 경험치 테이블 (`Current Level * 10`)
- 능력치 업그레이드 비용 로직 (n레벨 업그레이드 비용 = n 포인트)
- `ui/`: 능력치 확인 패널

**완료 기준**: 적 처치 시 EXP를 획득하고, 레벨업 시 스탯 포인트가 지급되어 5대 기관에 투자할 수 있다.

---

### Phase 6a: 장비 시스템 (Equipment)

**목표**: 무기/방어구/액세서리 슬롯 및 장착 효과 구현.

**작업 범위**:
- `item/`: 무기/방어구/액세서리 데이터 구조 및 등급별 스탯 (`CreateWeapon` 등)
- `battle/`: 장착 장비의 전투 보너스 적용 (손 스탯 연동, 무기별 특수 효과)
- 아이템 드롭, 월드 아이템 배치 및 획득
- `ui/`: 인벤토리 및 장착 화면

**완료 기준**: 적 처치 시 아이템이 드롭되고, 플레이어가 이를 획득하여 장착하면 전투 스탯에 반영된다.

---

### Phase 6b: 유물 및 소모품 (Relics & Consumables)

**목표**: 전역 효과를 가지는 유물 시스템 및 소모품 구현.

**작업 범위**:
- `item/`: 유물 데이터 구조, 전역 효과 정의 (예: Chicken Game, Scope 등)
- `battle/`: 유물 효과의 전투 계산 반영
- 소모품 사용 로직 (HP/SP 회복, 버프)
- 상점 시스템 (`OpenShop`, `BuyItemFromShop`, `SellItemToShop`)

**완료 기준**: 유물 장착 시 모든 오퍼레이터에게 전역 효과가 적용된다. 상점에서 소모품을 구매하고 사용할 수 있다.

---

### Phase 7: 게임 진행 및 층 구조 (Progression)

**목표**: 섹터 개념 및 층 진행 구조 (일반 → 보스 → 휴식) 구현.

**작업 범위**:
- `map/`: `FloorManager`, 층 타입별 맵 생성 (일반/보스/휴식)
- 층 전환 로직 및 플레이어 위치 재배치
- 휴식 층 텍스트 UI 인터페이스 (상점 연동)
- 모집 티켓 시스템 (랜덤 오퍼레이터 영입)

**완료 기준**: 4개 일반 층 → 보스 층 → 휴식 층 순으로 진행되며, 각 층 타입에 맞는 맵과 UI가 표시된다.

---

### Phase 8: 콘텐츠 - 기근 레이어 (Content: Famine)

**목표**: 첫 번째 레이어 '기근(Famine)'의 적, 엘리트, 보스 배치.

**작업 범위**:
- `entity/`: 엘리트 적 구조 (1.5배 스탯, 스킬 1개, EXP 2배)
- `battle/`: 보스 '아몬(Amon)' 특수 패턴 (스킬 3개)
- 적 스폰 시 엘리트 교체 확률 (5%)
- 적 종류별 드롭 및 골드 보상 적용

**완료 기준**: 기근 레이어의 모든 적 종류가 스폰되고, 보스 층에서 아몬과 전투가 가능하다.

---

### Phase 9: 폴리싱 및 밸런싱 (Polishing)

**목표**: 상태이상 완성, UI 개선, 전반적인 게임 밸런싱.

**작업 범위**:
- `battle/`: 분노/굶주림/병약 상태이상 완성
- `ui/`: 상태이상 표시, 전체 UI 정리
- 수치 밸런싱 (적 스탯, 아이템 드롭률, 골드 수급)
- NPC 퀘스트 구조 (섹터당 1명)

**완료 기준**: 전체 게임 루프가 처음부터 끝까지 오류 없이 진행된다.

---

## 단계 의존 관계 요약

```
Phase 1 (렌더링)
    └─ Phase 2 (맵 + 엔티티)
            └─ Phase 3 (전투)
                    ├─ Phase 4 (AI)
                    └─ Phase 5 (성장)
                            └─ Phase 6a (장비)
                                    └─ Phase 6b (유물/소모품)
                                            └─ Phase 7 (층 진행)
                                                    └─ Phase 8 (콘텐츠)
                                                            └─ Phase 9 (폴리싱)
```
